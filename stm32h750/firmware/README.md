# Kestrel H750 firmware (STM32CubeIDE project)

The complete application demoed in the submission: OV2640 camera → motion
gate (skip) → st_yololcv1 192 INT8 person detection (X-CUBE-AI) → LCD HUD,
with STOP-mode sleep/wake and UART/RAM telemetry. Runs from QSPI flash at
`0x90000000` behind the WeAct `11-ExtMem_Boot_USB` bootloader.

## Building

1. STM32CubeIDE 2.0.0+, STM32Cube FW_H7 V1.13.0, **X-CUBE-AI 10.2.1 pack
   installed** (the `.cproject` references the pack's `Middlewares/ST/AI`
   include path and `NetworkRuntime1020_CM7_GCC.a` under the default
   CubeMX repository location `C:/Users/<you>/STM32Cube/Repository/Packs/...`;
   adjust those two paths in Project Properties if yours differs).
2. Import: File → Open Projects from File System → this folder.
3. Build (Debug config) → produces `08-DCMI2LCD.hex` linked for QSPI.
4. Flash: hold K1 → plug USB → release at slow LED blink →
   `WeAct_HID_Flash_NETFW.exe` (WeAct MiniSTM32H7xx SDK) → select the hex.

## Caution: regenerating with CubeMX

CubeMX code generation silently reverts two lines in
`Src/main.c:SystemClock_Config` every time: re-add
`__HAL_PWR_VOLTAGESCALING_CONFIG(PWR_REGULATOR_VOLTAGE_SCALE0);` and change
`FLASH_LATENCY_0` back to `FLASH_LATENCY_1`. Also never raise HCLK above
120 MHz (QSPI execute-in-place ceiling) and keep the AXI-SRAM MPU region
`NOT_SHAREABLE` (Shareable disables the D-cache on the single-core M7).

Kestrel-specific sources (regen-proof, outside CubeMX ownership):
`detection.c/h`, `img_preproc.c/h`, `ai_infer.c/h`, `gate.c/h` (copy of
`modules/motion_gate`), `uart_log.c/h`, `stop_mode.c/h`, `temp_sensor.c/h`,
plus the `network*.c/h` files generated by `stedgeai generate`.
